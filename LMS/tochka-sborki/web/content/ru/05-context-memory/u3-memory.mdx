> 💡 **Не только Claude Code.** `CLAUDE.md` — конкретная реализация паттерна «persistent context file». В Aider это `.aider.conf.yml` + `CONVENTIONS.md`, в Hermes — DeepVista skill schema, в Cline — `.clinerules`. Концепция универсальная: контекст лежит в файле, агент читает при старте сессии.

<Phase type="activation">

Когда ты начинаешь новую сессию с Claude Code — что ты каждый раз объясняешь заново?

Не думай об этом как об «инструкциях» — представь это как **годовые кольца дерева**: каждый сезон оставляет след прямо в ткани древесины — засуха, рост, морозы — всё вписано в структуру, не в отдельный архив. Дерево не «помнит» прошлый год — оно им сделано.

Прокрути в голове свою работу с агентом: что из твоего опыта уже «вросло» в структуру проекта? Что пока ещё не закреплено и каждый раз произносится устно, как будто впервые?

</Phase>

<Phase type="reflection">

Каждый раз объяснять одно и то же — это потеря времени и токенов.

Но есть и обратная проблема: CLAUDE.md с тысячей строк хуже, чем CLAUDE.md на 100 строк. Чем длиннее — тем хуже агент следует инструкциям.

Не думай об этом как о «коротком конспекте» — представь себя **ткачом у станка**: нить основы натянута чётко и напряжённо — именно это натяжение держит всю ткань. Если добавить слишком много нитей сразу, ткань провиснет, петли запутаются.

Разгляди свою систему памяти: где сейчас натяжение правильное, а где — уже провисает под лишними нитями? Что держит форму, а что разрыхляет структуру?

</Phase>

<Phase type="concept">

## Три файла памяти

### CLAUDE.md — контекст проекта (главный)

```markdown
# Project Context

## Моя роль
[Профиль, экспертиза]

## Проект
[Что это, зачем нужно]

## Стек
- Node.js 18, TypeScript, React

## Структура
- /src — код
- /tests — тесты

## Как запустить
npm install && npm run dev

## Что делегировать
- Написание кода, тесты, документация

## Что решаю сам
- Архитектура, финальный review
```

**Лимит: ~200 строк.** Больше — хуже следование.

### TODO.md — текущие задачи

```markdown
## 🔴 СЕЙЧАС
- [ ] Задача 1 (2 ч)

## 🟡 СЛЕДУЮЩИЕ
- [ ] Задача 2

## ✅ ГОТОВО
- [x] Задача 3
```

### AGENTS.md — инструкции для агента

```markdown
## Для любой задачи
- Используй TypeScript
- Пиши тесты вместе с кодом

## Для /explore
- Фокусируй на архитектуре

## Чего избегать
- Не меняй архитектуру без согласования
```

### MEMORY.md — автоматическая память Claude

В отличие от трёх файлов выше, MEMORY.md **пишет Claude сам**.

Что сохраняет: паттерны, исправления, особенности проекта.
Где хранится: `~/.claude/projects/<hash>/memory/` — только локально.
Управление: `/memory` — посмотреть и изменить.

</Phase>

<Phase type="practice">

Создай свою систему памяти:

<OsBlock os="mac">

```bash
mkdir -p ~/my-ai-project
cd ~/my-ai-project
touch CLAUDE.md TODO.md AGENTS.md
```

</OsBlock>

<OsBlock os="windows">

```powershell
New-Item -ItemType Directory $HOME\my-ai-project -Force
cd $HOME\my-ai-project
ni CLAUDE.md, TODO.md, AGENTS.md -ItemType File
```

</OsBlock>

Заполни CLAUDE.md по шаблону выше — используй реальный проект или задачу из своей работы.

Добавь в TODO.md хотя бы 3 задачи, которые у тебя реально есть прямо сейчас.

Запусти Claude Code и проверь: `«Что ты знаешь о моём проекте?»`

Сохрани ответ в `my-experiments/u3-memory-test.md`

</Phase>
