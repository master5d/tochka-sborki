<Phase type="activation">

Представь, что у тебя есть **джинн** — и ты торгуешься с ним об условиях желания. Он исполняет ровно то, что сказано — ни словом больше, ни словом меньше. Хитрец: если в формулировке лазейка — он её найдёт.

Прокрути в голове задачу, которая у тебя повторяется каждую неделю — отчёт, подготовка к встрече, анализ, контент, ответы. Поймай её. Теперь представь, что джинну нужно сформулировать это желание так, чтобы он не мог выкрутиться. Что именно надо уточнить — чтобы ни одна лазейка не осталась?

</Phase>

<Phase type="reflection">

Теперь другой образ: ты **режиссёр**, который разбирает мизансцену перед первой репетицией.

Удержи в голове свою повторяющуюся задачу как сцену: кто в ней играет, какие реплики, какой финал акта. Режиссёр не надеется, что актёры сами догадаются — он расставляет позиции заранее.

Разгляди: какие именно «позиции» в этой задаче ты никогда не расставлял AI? Что останется за кулисами, если их не прописать?

</Phase>

<Phase type="concept">

## Мета-промптинг: промпт, который пишет промпты

Если не знаешь, с чего начать — попроси Claude написать промпт за тебя:

```
Ты — эксперт в составлении промптов для языковых моделей.

Напиши детальный промпт для ассистента, который будет [описание задачи].

Каждый его ответ должен:
- [критерий 1]
- [критерий 2]
- [критерий 3]

Включи все параметры, важные для качества.
```

### Шаблон переиспользуемого ассистента

Для задач, которые повторяются (Claude Project, Gemini Gem, Claude Code Skill):

```
Role:         кто этот ассистент (экспертиза, стиль, границы)
Objective:    главная цель — зачем он существует
Context:      в какой ситуации его используют
Instructions: список правил, по которым он работает
Notes:        напоминания, edge cases, запреты
```

### Чеклист перед отправкой

- [ ] Роль ясна (кто отвечает с какой экспертизой)?
- [ ] Задача конкретна (что точно нужно)?
- [ ] Output определён (как узнать, что правильно)?
- [ ] Ограничения понятны (что не делать)?
- [ ] Нет дистракторов (нет лишнего текста)?
- [ ] Важное в начале (не в середине)?

</Phase>
