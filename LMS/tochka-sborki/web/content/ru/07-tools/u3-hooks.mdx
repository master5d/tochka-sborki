<Phase type="reflection">

Если ты делаешь что-то после каждого действия агента — это Hook.

Не думай об этом как о «скрипте». Представь цепочку домино: ты толкаешь первую костяшку — и весь ряд проходит сам, одна за другой, без твоего участия. Событие в Claude Code — первая костяшка; Hook — это весь последующий ряд.

Разгляди в голове свою цепочку: сколько «костяшек» сейчас требуют, чтобы ты их толкал по одной?

</Phase>

## События для Hooks

| Событие | Когда срабатывает |
|---------|-----------------|
| `PreToolUse` | До того как агент использует инструмент |
| `PostToolUse` | После того как агент использует инструмент |
| `Notification` | Когда агент хочет привлечь внимание |
| `Stop` | Когда агент завершил работу |

## Конфигурация Hook

В `.claude/settings.json`:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "echo 'Файл записан: ' && date >> ~/claude-log.txt"
          }
        ]
      }
    ]
  }
}
```

## Практические примеры

**Логирование:** каждый раз когда агент пишет файл — сохранять в лог

**Уведомления:** когда агент завершил задачу — отправить сообщение в Telegram

**Валидация:** перед тем как агент запускает bash-команду — проверить что она безопасна

**Форматирование:** после того как агент написал TypeScript файл — запустить prettier

## Безопасность

Hook может запускать любую команду. Несколько правил:
- Не запускай hooks из непроверенных источников
- Проверяй что hook делает перед добавлением
- Для production-систем ограничивай что агент может делать через PreToolUse

Напиши свой первый Hook — простой, который логирует активность:

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date): Claude завершил задачу\" >> ~/claude-activity.log"
          }
        ]
      }
    ]
  }
}
```

Добавь в `.claude/settings.json`, запусти Claude Code, дай любую задачу.

Проверь лог: `cat ~/claude-activity.log`
