<Phase type="activation">

Представь промпт, к которому ты возвращаешься снова и снова — проверка кода, summary встречи, release notes.

Не описывай его как «текст». Представь ловчую птицу сокольника: она не думает каждый раз, как атаковать — она натренирована на одну команду и бросается на добычу мгновенно. Твой повторяющийся промпт — это уже почти та птица. Прокрути в голове: на какую именно «добычу» она кидается?

**Сколько раз ты уже давал ей эту команду вручную?**

</Phase>

<Phase type="reflection">

Хороший переиспользуемый промпт — это уже почти Skill.

Не думай об этом как о «сохранённом тексте». Представь слесаря, который вырезал отмычку точно под конкретный замок: ключ входит без примерки, с первого движения — никакого подбора, никакого трения. Skill — это такая же отмычка для твоей задачи: `/skill-name`, и агент открывает ровно то, что нужно.

Разгляди в голове: какой замок в твоей работе пока открывается только подбором наугад?

</Phase>

<Phase type="concept">

## Что такое Skill

Skill — это Markdown-файл с инструкцией, который Claude Code выполняет при вызове `/название`.

### Структура файла skill'а

```markdown
---
name: meeting-summary
description: Суммирует встречу из транскрипта или заметок
---

# Meeting Summary Skill

## Цель
Создать структурированное резюме встречи.

## Инструкция
Ты — ассистент по встречам. На основе предоставленных данных:

1. Выдели главные решения (3–5 пунктов)
2. Определи задачи с ответственными и сроками
3. Отметь открытые вопросы

## Формат вывода
### Решения
- [список]

### Задачи
- [ ] [задача] — [ответственный] — [срок]

### Открытые вопросы
- [список]
```

### Где хранить Skills

- `~/.claude/skills/` — личные, работают во всех проектах
- `.claude/skills/` — проектные, в git

### Как вызывать

```
/meeting-summary [вставь транскрипт или заметки]
```

## Plugin vs Skill

| | Plugin | Skill |
|--|--------|-------|
| Что это | Пакет с несколькими skills | Одно конкретное действие |
| Как использовать | Установить один раз | Вызвать через `/название` |
| Пример | `firecrawl@local` | `/firecrawl:firecrawl-cli` |

</Phase>

<Phase type="practice">

Создай skill для задачи из Активации:

<OsBlock os="mac">

```bash
mkdir -p ~/.claude/skills
touch ~/.claude/skills/my-first-skill.md
```

</OsBlock>

<OsBlock os="windows">

```powershell
New-Item -ItemType Directory "$HOME\.claude\skills" -Force
ni "$HOME\.claude\skills\my-first-skill.md" -ItemType File
```

</OsBlock>

Заполни по шаблону выше. Используй реальный промпт который у тебя уже есть.

Протестируй:
```
/my-first-skill [дай реальные данные]
```

Сохрани файл skill'а в `my-experiments/u4-my-skill.md`
