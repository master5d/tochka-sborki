<Phase type="activation">

Ты прошёл через 4 unit'а и сделал выбор стека. Этот последний unit — **твой план миграции** на 7 дней.

Прежде чем двигаться вперёд — не думай об этом как о «смене инструмента». Представь **переезд в новый дом**: никто в здравом уме не выкидывает старую мебель до того, как въедет в новое жильё. Сначала ты перевозишь самое нужное, живёшь в двух местах неделю, потом отдаёшь ключи. А ещё точнее — это смена лошадей на переправе: посреди реки не спрыгивают с одной и не плывут к другой вплавь, пересаживаются **шаг за шагом, держась за обеих**.

Удержи в голове: что из твоего текущего workflow — «старая мебель» (можно перевезти потом), а что — «документы и ноутбук» (нужно сразу на новом месте)?

</Phase>

<Phase type="reflection">

Представь, как змея меняет кожу 🐍. Она не срывает старую одним движением — новая нарастает под ней, старая сходит постепенно, и какое-то время обе существуют одновременно. Это не слабость — это единственный способ не остаться без покрова.

Поймай ту же мысль здесь. Самая частая ошибка миграции: «удалю Claude Code, поставлю Aider, и сразу буду эффективнее». Не работает — новая кожа ещё не окрепла.

Реалистичный путь — это **линька**:
- **Дни 1–2:** новая кожа нарастает под старой. Новый стек — дополнение, старый работает.
- **Дни 3–4:** старая начинает сходить. Одна реальная задача уходит на новый стек — замечаешь, где медленнее, где быстрее.
- **Дни 5–7:** решаешь — hybrid навсегда, или старая кожа уже отпала сама.

</Phase>

<Phase type="concept">

## Шаблон плана миграции

```markdown
# Миграция на [твой стек] — 7 дней

## День 1: Setup
- [ ] Установить Frontend agent (Hermes / Aider / Cline / Claude Code)
- [ ] Поднять LiteLLM gateway локально
- [ ] Получить API keys (Cerebras / Gemini / etc.)
- [ ] Сделать «hello world» запрос через curl

## День 2: Reproduce
- [ ] Взять ОДНУ задачу, которую ты решил на старом стеке за последнюю неделю
- [ ] Повторить её на новом стеке
- [ ] Записать: где медленнее / где быстрее / где проблема

## День 3–4: Real work
- [ ] Сделать одну реальную рабочую задачу на новом стеке
- [ ] Старый стек оставить как fallback
- [ ] Не уговаривай себя что «новое лучше» — фиксируй факты

## День 5: Customize
- [ ] Настрой 1–2 skill / hook под свой workflow
- [ ] Добавь свой CLAUDE.md / AGENTS.md для нового агента
- [ ] LiteLLM: настрой fallback chain

## День 6: Honest comparison
- [ ] Запиши: что лучше на новом стеке, что хуже
- [ ] Где hybrid имеет смысл (например, Claude Code для дизайна + Aider для рефакторинга)

## День 7: Decision
- [ ] Решение: полная миграция / hybrid / возврат
- [ ] Если возврат — что нужно от стека чтобы передумать?
- [ ] Если миграция — закрепляй: убираешь старый stack из активной памяти
```

</Phase>
