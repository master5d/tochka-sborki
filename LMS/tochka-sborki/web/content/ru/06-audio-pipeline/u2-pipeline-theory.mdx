<Phase type="activation">

Вспомни один информационный поток, который повторяется каждую неделю — он живёт в тебе уже как привычка.

Не воспринимай его как «рутину». **Представь его как кровеносную систему:** сердце (trigger) сжимается и гонит кровь по кругу; лёгкие (обработка) насыщают её кислородом; капилляры (доставка) доносят результат до каждой клетки.

Прокрути этот круг в голове: где сердце твоего потока — что его запускает? И куда кровь добирается в конце?

</Phase>

<Phase type="reflection">

Представь, что дирижёр раскрывает партитуру и видит: первые скрипки играют одно, трубы — другое, ударные — третье. Каждая секция знает свой вход. Никто не ждёт указания «теперь ты».

**Разгляди свой еженедельный поток так же:** какие «голоса» в нём уже знают свою партию (ты делаешь это автоматически), а какой инструмент всё ещё нуждается в том, чтобы ты лично вступал с дирижёрской палочкой?

Та секция, где ты стоишь с палочкой вручную — это и есть место для pipeline.

</Phase>

<Phase type="concept">

## Три типа pipeline

### 1. Ручной pipeline (сегодня)
Ты запускаешь вручную когда нужно. Идеально для нерегулярных задач.

```
Ты → [URL/файл] → Claude → результат
```

### 2. Schedule pipeline
Запускается по расписанию. Подходит для регулярных задач.

```
Каждое утро в 8:00 → собрать RSS → анализировать → отчёт в Slack
```

Инструменты: n8n, Make.com, GitHub Actions, Claude Code Hooks.

### 3. Event-driven pipeline
Запускается когда что-то происходит.

```
Новый файл в папке → автоматически транскрибировать → суммировать
Новый email → классифицировать → черновик ответа → на согласование
```

## Инструменты для pipeline

| Инструмент | Уровень | Для чего |
|------------|---------|---------|
| Claude (chat) | Начало | Разовые задачи, тестирование |
| Claude Code | Средний | Работа с файлами, проектами |
| Make.com / n8n | Продвинутый | Автоматизация без кода |
| Claude Code Hooks | Экспертный | Событийная автоматизация |

> 💡 Начни с ручного pipeline. Как только убедишься что алгоритм работает — автоматизируй trigger.
