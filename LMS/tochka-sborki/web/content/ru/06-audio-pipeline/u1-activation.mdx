<Phase type="activation">

Вспомни один цикл последних трёх дней: взял информацию из сети — и куда-то её доставил.

Не представляй это как «работу с компьютером». **Представь это как конвейер:** деталь (информация) ложится на ленту → проходит станцию за станцией (читаешь, выделяешь, пересказываешь) → выходит готовым изделием на другом конце.

Прокрути этот конвейер в голове: сколько станций? На какой из них лента замедляется и начинает копиться очередь?

</Phase>

<Phase type="reflection">

Удержи карту этого маршрута перед глазами — как картограф, который только что прошёл местность пешком.

На каждом повороте он ставит отметку: «здесь болото» (трудно), «здесь твёрдая дорога» (легко). **Разгляди свой маршрут так же:** какие участки — болото ручного труда, а какие уже идут почти сами?

Твой pipeline сегодня — это первая карта, которую не нужно проходить вручную дважды. Какой участок ты нанесёшь на неё первым?

</Phase>

<Phase type="concept">

## Любая автоматизация = Trigger → Action

Это фундаментальный mental model:

| Trigger | Action |
|---------|--------|
| Будильник 6:00 | Телефон звонит |
| Оплата прошла | Email + SMS клиенту |
| Ты даёшь URL агенту | Агент скрапит → анализирует → сохраняет результат |

### Три вопроса перед построением pipeline

1. **Trigger:** что запускает процесс? (ты вручную, schedule, новый файл, webhook)
2. **Action:** что должно произойти? (scrape → analyze → save → notify)
3. **Condition:** когда НЕ запускать? (дубли, rate limits, ошибки)

### Наш pipeline сегодня

```
Trigger:  Ты даёшь URL статьи
   ↓
Action 1: Firecrawl скрапит страницу → получает чистый текст
   ↓
Action 2: Claude анализирует текст
   ↓
Action 3: Результат сохраняется в Markdown-файл
```

Это ручной trigger. Но принцип тот же — что для Zapier, что для Claude Code Hooks.

</Phase>
