<Phase type="concept">

## A professional prompt structure: CTID

### Quick mnemonic to get started

```
Context     = who you are and what's the situation
Task        = what exactly needs to be done
Instructions = how, in what tone, in what format
Data        = examples, links, input information
```

### Full structure for reproducible results

```markdown
## Role
You are — [expertise and specialization]

## Task
[Specific goal — one sentence]

## Inputs
[What you're providing: files, URLs, data, context]

## Expected output
[Format, structure, length, quality criteria]

## Constraints
- Don't do: [X]
- Use only: [Y]
- Max: [Z]
```

### Example in action

```markdown
## Role
You are a research agent specializing in analyzing
technical docs and extracting key insights.

## Task
Analyze the API documentation and create a short reference
with examples for intermediate Python developers.

## Expected output
A Markdown file with this structure:
1. API description (2–3 sentences)
2. Main endpoints (table)
3. Authentication
4. 3 practical code examples

## Constraints
- Examples must be complete (copy-paste runnable)
- Length: max 500 lines
- Use Python 3.10+
```

> The more precise the spec, the fewer iterations. Investing in the prompt saves time many times over.

</Phase>
