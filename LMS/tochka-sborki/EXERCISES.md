## 📝 Упражнение 1: Переписать команду в делегирование

**Цель:** Научиться переходить от команд к описанию результата.

### Задание:

У тебя есть вот такой "промпт" (команда):

```
Установи npm
Создай package.json
Установи React
Установи Tailwind
Создай папку src
Создай App.tsx
```

### Что делать:

1. Переписать это в **делегирование** (описание результата)
2. Сохрани в `my-experiments/exercise-1-delegation.md`

### Пример результата:

```markdown
# Exercise 1: Delegation

## Вход (ЧТО БЫЛО):
Команды по шагам...

## Выход (ЧТО СТАЛО):
Я хочу создать React приложение для [НАЗНАЧЕНИЕ].

Требования:
- TypeScript поддержка
- Tailwind CSS
- Структура: компоненты в src/

Результат: приложение, которое запускается на localhost:3000
```

### Проверка:

- [ ] Переписал "Установи X" в "Мне нужно Y"?
- [ ] Добавил цель и контекст?
- [ ] Описал, как узнать, что это работает?

## 🔄 Упражнение 2: Экспериментирование с промптами

**Цель:** Научиться итерировать и сравнивать результаты.

### Задание:

Создай компонент кнопки. Но сделай это в 3 итерациях:

**Попытка 1:** Минимальный промпт
```
Создай React компонент кнопки.
```

**Попытка 2:** С деталями
```
Создай React компонент кнопки.
- Поддерживает primary/secondary варианты
- На hover меняется цвет
- Disabled состояние
```

**Попытка 3:** Полный промпт
```
Создай React компонент Button для дизайн-системы.

Требования:
- Варианты: primary (синяя), secondary (серая), danger (красная)
- Размеры: sm, md, lg
- Состояния: normal, hover, active, disabled
- На hover: scale 1.05
- Disabled: opacity 0.5
- TypeScript, с типизацией props

Результат: файл Button.tsx с примерами использования
```

### Что делать:

1. Запусти Claude Code в папке `my-experiments/button-exercise`
2. Попробуй все 3 варианта промпта
3. Сравни результаты
4. Сохрани в `exercise-2-iterations.md`

### Сохранить:

```markdown
# Exercise 2: Iterations

## Попытка 1: Минимальный промпт
[Результат от Claude Code]

## Попытка 2: С деталями
[Результат от Claude Code]

## Попытка 3: Полный промпт
[Результат от Claude Code]

## Анализ:
- Какой вариант был самый эффективный?
- Сколько было итераций уточнения?
- Какие детали были самыми важными?
```

### Проверка:

- [ ] Пробовал 3 варианта?
- [ ] Сравнил результаты?
- [ ] Выписал, какой вариант был лучше и почему?

## 📋 Упражнение 3: Структурирование контекста

**Цель:** Научиться создавать хорошие CLAUDE.md и TODO.md файлы.

### Задание:

Возьми существующий проект (любой, который есть на диске). Создай для него:

1. **CLAUDE.md** - полный контекст проекта
2. **TODO.md** - текущие задачи
3. **AGENTS.md** - инструкции для Claude Code

### Структура CLAUDE.md:

```markdown
# Project Context

## Моя роль
[Кто ты для этого проекта]

## Проект
[Что это, зачем нужно]

## Стек
[Технологии и версии]

## Структура
[Папки и их назначение]

## Как запустить
[Команды для старта]

## Текущее состояние
[Что работает, что нет]

## Когда вызывать Claude Code
[Для каких задач]
```

### Что делать:

```bash
cd [твой-проект]
touch CLAUDE.md TODO.md AGENTS.md

# Заполни каждый файл
# Сохрани результаты в git
git add CLAUDE.md TODO.md AGENTS.md
git commit -m "Add context files for Claude Code"
```

### Проверка:

- [ ] CLAUDE.md содержит всю необходимую информацию?
- [ ] TODO.md расставлены приоритеты?
- [ ] AGENTS.md описывают твой стиль работы?
- [ ] Файлы добавлены в git?

## 🎯 Упражнение 4: Использование Explore

### Задание:

Запусти `/explore` в существующем проекте с разными параметрами:

```bash

```

[Результаты]

- "Какие файлы отвечают за auth?" → [Ответ]
- "Как работает..."

### Проверка:

- [ ] Использовал все 3 уровня thoroughness?

## 🧪 Упражнение 5: Мета-промптинг

**Цель:** Научиться создавать промпты, которые создают промпты.

### Задание:

Напиши мета-промпт для своего проекта. Это промпт, который подскажет Claude, как создавать промпты для ТВОЕГО проекта.

### Пример:

```
Создай промпт для анализа кода в моём [ПРОЕКТ] проекте.

Промпт должен:
1. Просить агента сначала понять архитектуру
2. Затем анализировать конкретный файл
3. Предлагать улучшения, учитывая контекст проекта
4. Давать примеры кода

Убедись, что промпт ясен и использует информацию из CLAUDE.md
```

### Что делать:

1. Напиши мета-промпт
2. Дай его Claude Code
3. Получи результирующий промпт
4. Используй этот промпт для реальной работы
5. Сохрани в `exercise-5-meta-prompt.md`

### Результат:

```markdown
# Exercise 5: Meta-prompting

## Мой мета-промпт:
[Что я просил создать]

## Результирующий промпт от Claude:
[Промпт, который Claude создал]

## Как я это использовал:
[На какую задачу применил]

## Результат:
[Работало ли это?]
```

## 🎨 Упражнение 9: Vibe Coding — MVP за вечер

**Цель:** Пройти полный цикл от идеи до рабочего продукта используя AI-стек.

### Что такое Vibe Coding

Это подход, при котором ты фокусируешься на **продуктовом мышлении**, а не на технической реализации. AI-инструменты покрывают код, ты — задаёшь направление.

### Стек: Perplexity → Grok → Lovable → Supabase

| Инструмент | Роль | Что делает |
| :--- | :--- | :--- |
| **Perplexity** | Research | Анализ рынка, конкуренты, целевая аудитория |
| **Grok** | Planning | Пошаговые инструкции, генерация промптов |
| **Lovable / bolt.new** | Frontend | Полный UI по промпту (React + Tailwind) |
| **Supabase** | Backend | База данных, авторизация, API без кода |

### Задание: Построй инструмент для себя

Выбери задачу из своей жизни (не абстрактную!). Примеры:
- Трекер привычек
- Дашборд для своего проекта
- Простая CRM для клиентов
- Генератор отчётов

### Шаги:

**Шаг 1: Research (Perplexity, 20 мин)**
```
Запрос: "Что уже существует в категории [ТВОЯ ИДЕЯ]?
Топ-5 конкурентов, их слабые стороны, незакрытые потребности аудитории."
```

**Шаг 2: Plan (Grok, 15 мин)**
```
Запрос: "Я хочу построить [ИДЕЯ]. Дай пошаговый план MVP.
Составь промпт для Lovable/bolt.new — что именно ему написать."
```

**Шаг 3: Build Frontend (Lovable / bolt.new, 30 мин)**
```
Вставь промпт из шага 2. Итерируй — объясняй на языке дизайна:
"Сделай кнопку крупнее", "Добавь таблицу с данными", "Добавь боковое меню"
```

**Шаг 4: Connect Backend (Supabase, 20 мин)**
- Создай проект на supabase.com
- Lovable умеет подключиться напрямую через `/integrate supabase`
- Укажи таблицы — и данные сохраняются автоматически

### Результат

Сохрани в `my-experiments/exercise-9-vibe-build.md`:

```markdown
# Exercise 9: Vibe Coding

## Идея: [Что я строил]

## Research (Perplexity):
[Что узнал о рынке]

## Plan (Grok):
[Промпт, который использовал для сборки]

## Frontend (Lovable / bolt.new):
[Скриншот или описание UI]

## Backend (Supabase):
[Таблицы, что сохраняется]

## Итог:
- Сколько времени заняло: X часов
- Что работает end-to-end: да / нет
- Главный инсайт:
```

### Проверка:

- [ ] Выбрал реальную задачу, не абстрактную?
- [ ] Прошёл хотя бы 3 из 4 шагов стека?
- [ ] Frontend отображает данные?
- [ ] Есть хотя бы один рабочий user flow?
- [ ] Сохранил рефлексию в exercise-9?
