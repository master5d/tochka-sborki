# Blog as a living knowledge graph — design pass (NOT built)

**Тикет:** `fb_10b3d7deb968` (feature, area blog, impact 6) — блог как живой knowledge graph (evergreen notes) с визуалом и лендинг-страницей.
**Дата:** 2026-06-18
**Статус:** design-pass. **Заблокировано:** визуальная дирекция владельца (стиль графа) + контент-решение (evergreen-структура заметок).

## Контекст / что уже есть
Блог (`blog/`) = `lib/posts.ts` реестр с `related: string[]` (связи post→post, сейчас частично заполнены) + `tags`. Это **уже граф-данные** — узлы (посты) + рёбра (`related`) + кластеры (`tags`). Серия «Горизонты». Не хватает: (1) визуализации графа, (2) evergreen-структуры (атомарные заметки vs длинные посты), (3) лендинг-страницы графа.

## Подход (когда разблокируют) — фазами
**Фаза A — граф из существующих данных (низкий риск, без нового контента):**
- Pure-билдер `lib/graph.ts`: `buildGraph(posts) → { nodes:[{slug,title,tags}], edges:[{from,to}] }` из `related[]` (взаимные/односторонние). Тестируемо.
- Страница `/blog/graph` (или `/blog/map`): лёгкая SVG/canvas-визуализация (узлы=посты, рёбра=related, цвет=tag-кластер), клик→пост. **Без тяжёлых граф-либ** — простой force-light или радиальный layout (static export → детерминированный layout на билд-тайме, не рантайм-симуляция).
- На каждом посте: «Связанные» уже есть; добавить мини-«ты здесь» фрагмент графа.

**Фаза B — evergreen notes (контентная переработка):**
- Ввести атомарные evergreen-заметки (короче постов, одна идея) как отдельный тип в `posts.ts` (`kind: 'note' | 'post'`); заметки плотно линкуются. Это **контентная работа** владельца.

## Решения за владельцем (разблокировка)
1. **Визуальная дирекция** — стиль графа (минималистичный SVG в бренд-палитре? force-directed? радиальный? созвездие?). Нужен арт-ориентир (можно через brainstorming visual companion).
2. **Evergreen-модель** — переходить ли на атомарные заметки (Фаза B), или граф только по существующим постам (Фаза A)?
3. **Где живёт** — отдельная `/blog/graph` страница или интегрировано в индекс блога?

## Guardrails
- Static export: layout графа считается на билд-тайме (детерминированно), не рантайм-физика (иначе тяжело + нестабильно). Доступность: граф — прогрессивное улучшение, список постов остаётся базой.
- Не плодить граф-зависимости (d3-force тяжёл) — начать с собственного лёгкого layout.

## Вне scope (сейчас)
- Интерактивная рантайм-физика, zoom/pan-сложность.
- Авто-извлечение связей через эмбеддинги (можно позже — ср. [[project_embedding_agent]]).

## Готовность к build
Фаза A (граф из `related[]`) спроектирована и низкорисковая; нужен визуальный go. Тогда: `lib/graph.ts` (TDD) → `/blog/graph` страница (детерминированный layout) → blog build гейт.
