# Блог-серия «Горизонты» + первый пост `/blog/horizons`

**Дата:** 2026-05-31
**Статус:** на ревью
**Источник:** фидбэк студентки — настоящий барьер аудитории это *отсутствие воображения*
(«не знаю, как ещё можно / не знаю потолка»). Серия лечит именно его.

## Цель

Запустить рекуррентную серию **«Горизонты»** — что реально можно делать с AI, если ты
не технарь. Сквозной принцип = тезис пролога: **AI усиливает твой голос, а не подменяет
его.** Первый пост — карта/обзор возможностей; будущие посты бурят каждый горизонт вглубь.

## Архетип (не реальный человек)

«Умный творческий нетехнарь, который не знает потолка»: уже пользуется ChatGPT/Claude для
бытовых задач, доволен на этом уровне, но не представляет, что ещё бывает. Без имени, без
узнаваемых деталей конкретной студентки — обобщённый паттерн. Согласие не требуется.

## Первый пост — контент

- **Slug:** `horizons` → `https://mamaev.coach/blog/horizons/`
- **Title:** «Горизонты: что вообще можно делать с AI, если ты не технарь»
- **Голос:** тот же провокативно-тёплый градиент, что в прологе (2-е лицо, честно, тёплая
  ирония, без техно-поэзии и FOMO).
- **Открытие:** назвать ступор прямо — «Ты уже спрашиваешь ChatGPT про рецепты, гороскоп и
  тексты — и думаешь, что это и есть AI. Это как купить рояль и играть одним пальцем».
- **Меню горизонтов** (4 коротких конкретных виньетки, обобщённые из латентных кейсов
  аудитории + типовых):
  1. **Говорить с софтом, а не воевать с ним** — управлять сложным инструментом словами и
     метафорами, а не искать кнопку.
  2. **Привлекать клиентов, не становясь контент-заводом** — AI на операционке привлечения
     (ресёрч, черновики под правку, follow-up); голос остаётся твой.
  3. **Посадить рутину на агента** — агент ведёт повторяемое (админка, расписание, разбор
     почты), время уходит в ремесло.
  4. **Усилить ремесло, не подменив голос** — AI как усилитель экспертизы; явная граница:
     НЕ пишет за тебя.
- **Сквозная граница:** в каждом пункте помечено «это делает AI — а это остаётся твоим».
- **Финал:** тёплый, «ты не обязан знать это заранее — в этом и смысл»; ссылка на пролог.
  CTA-курс несёт `BlogFooter` (не врезаем в текст).
- **Длина:** ~6–7 мин.

## Имплементация (зеркалит структуру пролога; ноль риска для существующего)

- **Новый** `hub/components/blog/blog-prose.module.css` — брендовый лонгрид-базис (Geist-тело,
  Unbounded `h2`, неон `strong`, переделанная цитата без side-stripe, манифест-беты). По сути
  обобщение того, что мы сделали в `Prologue.module.css`, но как самостоятельный модуль.
- **Новый** `hub/components/blog/posts/horizons.tsx` — компонент поста: рендерит прозу внутри
  `.prose` из `blog-prose.module.css`. `locale: 'ru' | 'en'`, EN бросает
  `Error('EN translation pending')` (как секции пролога).
- **Новый** `hub/app/blog/horizons/page.tsx` — оборачивает `<Horizons />` в `PostLayout`,
  metadata + OG url `https://mamaev.coach/blog/horizons/`.
- `hub/lib/posts.ts`: добавить запись `horizons` (date `2026-05-31`, author, `~7 мин`, tags
  `['AI','кейсы','нетехнари','Точка Сборки']`, `related: ['prologue']`); в записи `prologue`
  проставить `related: ['horizons']` — взаимный кросс-линк «По теме».
- `PostLayout` остаётся generic — НЕ трогаем (значит `BlogFooter`, JSON-LD, «По теме»,
  back-links появляются у нового поста автоматически).
- `Prologue.module.css` НЕ трогаем (у пролога свой модуль) → исключаем конфликт стилей.

### Почему отдельный модуль, а не общий prose сразу

Общий `.prose`, навешенный в `PostLayout` на `{children}`, через element-селекторы
(`.prose p`, `.prose h2`) затронул бы и разметку пролога, у которого уже свои классы — риск
двойного стиля. Поэтому сейчас новый пост несёт свой модуль (как пролог — свой). Вынос общего
prose-базиса — отдельный безопасный проход позже.

## Вне рамок

- Глубокие посты по каждому горизонту (серия дальше).
- EN-перевод поста (`Error('EN translation pending')`).
- Вынос общего prose-стиля; content graph (Spec 2) — по-прежнему отложен.

## Верификация

- `npm run build` (hub) — чисто, `/blog/horizons` пререндерится, нет битых импортов.
- `vitest hub/lib/posts.test.ts` — зелёный (новый пост в реестре не ломает фильтр/сорт).
- Живой скрин `/blog/horizons/` на dev — читается как единый брендовый лонгрид, видны
  «По теме» (пролог) и `BlogFooter`; на `/blog/prologue/` теперь «По теме» → Горизонты.

## Процесс

Связная проза одной рукой (не сабагенты). После утверждения спеки — реализую, прогон
build + тестов, скрин.
