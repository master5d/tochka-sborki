<Phase type="activation">

Удержи в голове один из своих типичных промптов — тот, который ты отправляешь агенту снова и снова.

Не смотри на него как на «сообщение» — представь это как **папку с делом в руках следователя**: всё досье целиком — хронология, показания, улики — против одного брошенного вопроса вслед уходящему свидетелю. Первое даёт понимание, второе — только момент.

Разгляди свой промпт: что в нём — постоянное досье, а что — вопрос на ходу? Они живут в одном куске текста или по-прежнему перемешаны?

</Phase>

<Phase type="reflection">

Контекст — стабильный фон. Задание — конкретная инструкция на этот момент.

Не думай об этом как об «онбординге нового сотрудника» — представь, что ты **монтажёр фильма**: исходный материал огромен, но в финальный монтаж входит только то, что действительно работает на сцену прямо сейчас. Всё остальное — на полке, не в кадре.

Прокрути в голове: если бы твои промпты — это черновой монтаж, что ты вырезал бы в финале? Что могло бы жить на полке и браться только при необходимости?

</Phase>

<Phase type="concept">

## Разделение контекста и промпта

```
Контекст  = информация о мире (цели, роли, правила, данные, история)
Промпт    = конкретная инструкция на этот запрос

«Напиши письмо» = ПРОМПТ (задание)

«Я основатель стартапа, пишу инвестору, стиль лаконичный и уверенный,
цель: договориться о встрече в среду вечером» = КОНТЕКСТ
```

### Как разделять явно

```markdown
## КОНТЕКСТ
[Роль, цель, ограничения, данные — то что стабильно]

## ЗАДАЧА
[Конкретная инструкция для этого запроса]
```

Или через XML-теги:

```xml
<context>
Ты — специалист по code review. Проект использует Python 3.11.
</context>

<task>
Проверь этот код на ошибки безопасности.
</task>

<code>
[Код здесь]
</code>
```

### Где хранить стабильный контекст

| Что | Где хранить |
|-----|-------------|
| Правила проекта, стек | CLAUDE.md в папке проекта |
| Личные предпочтения | ~/.claude/CLAUDE.md |
| Текущие задачи | TODO.md |
| Инструкции для агента | AGENTS.md |

</Phase>

<Phase type="practice">

Возьми любую задачу из работы. Разбей её на два блока:

```markdown
## КОНТЕКСТ
[Кто ты, что за проект, стек, ограничения]

## ЗАДАЧА
[Только конкретная инструкция]
```

Сравни результат с тем, как ты обычно формулируешь промпт.

Сохрани в `my-experiments/u2-context-vs-prompt.md`
