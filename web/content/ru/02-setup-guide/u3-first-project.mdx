<Phase type="activation">

У тебя есть идея — проект, задача, что-то, что давно хотелось сделать.

Не представляй это как «создание папки». Представь, как каменщик кладёт **первый кирпич в кладке**: он не думает о всём доме сразу — он выравнивает только этот один, потому что именно от него пойдёт всё остальное.

Прокрути в голове: что у тебя сейчас **первый кирпич**? Не план. Не список. Один момент, с которого начнётся движение. **Удержи его.**

</Phase>

<Phase type="reflection">

Вспомни, как ты давал имя чему-то важному — проекту, питомцу, персонажу в игре. В момент, когда имя произнесено, — оно начинает определять форму.

Не думай об этом как «я создал CLAUDE.md». Представь, как **называют новорождённого**: имя ещё не отражает ничего, что будет, — но уже задаёт характер, задаёт ожидание, задаёт вектор.

Прокрути в голове: что ты сейчас «называешь»? **Какое имя даёшь своему проекту — и что это имя говорит агенту о том, кто ты и чего хочешь?**

</Phase>

<Phase type="concept">

## CLAUDE.md — память агента для проекта

CLAUDE.md — это Markdown-файл, который Claude Code читает автоматически при каждом запуске в папке.

### Минимальный шаблон

```markdown
# Project Context

## Моя роль
[Кто я, что я знаю, мой опыт]

## Проект
[Что это за проект, зачем нужен]

## Стек
- Node.js 18
- [другие технологии]

## Структура папок
- `/src` — основной код
- `/my-experiments` — эксперименты

## Как запустить
[Команды для старта]

## Что делегировать Claude
- Написание кода по ТЗ
- Тесты
- Документирование

## Что решаю сам
- Архитектурные решения
- Финальная проверка качества
```

### Создание проекта

<OsToggle />

<OsBlock os="mac">

```bash
mkdir -p ~/my-project
cd ~/my-project
git init
touch CLAUDE.md
mkdir -p src my-experiments
```

</OsBlock>

<OsBlock os="windows">

```powershell
New-Item -ItemType Directory $HOME\my-project -Force
cd $HOME\my-project
git init
ni CLAUDE.md -ItemType File
New-Item -ItemType Directory src, my-experiments -Force
```

</OsBlock>

### Советы по CLAUDE.md

- Держи до **200 строк** — длиннее снижает качество следования инструкциям
- Будь конкретным: `«Используй 2-пробельный отступ»` лучше чем `«форматируй правильно»`
- Обновляй когда меняется архитектура или стек

</Phase>

Создай свой первый проект:

<OsBlock os="mac">

```bash
mkdir -p ~/my-ai-project
cd ~/my-ai-project
git init
touch CLAUDE.md
mkdir -p my-experiments
```

</OsBlock>

<OsBlock os="windows">

```powershell
New-Item -ItemType Directory $HOME\my-ai-project -Force
cd $HOME\my-ai-project
git init
ni CLAUDE.md -ItemType File
New-Item -ItemType Directory my-experiments -Force
```

</OsBlock>

Заполни CLAUDE.md по шаблону — напиши про реальный проект или задачу, над которой хочешь работать.

Затем запусти Claude Code в этой папке:

```bash
claude
```

Задай агенту: `«Прочитай CLAUDE.md и скажи, что ты теперь знаешь о проекте»`

Сохрани результат в `my-experiments/u3-claude-reads-context.md`

</Phase>
