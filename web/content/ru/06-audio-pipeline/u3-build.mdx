<Phase type="activation">

У тебя есть дизайн pipeline из прошлого unit'а — набросок соединений между шагами.

Не смотри на него как на схему. **Представь его как сантехнику:** каждое соединение — фитинг, по которому идёт поток. Где трубы соединяются плотно, вода течёт. Где соединение наспех — капает.

Прокрути в голове каждый стык своего дизайна: какое из них, если честно, выглядит как временный хомут с изолентой? Это и есть место, которое нужно укрепить до сборки.

</Phase>

<Phase type="reflection">

Представь режиссёра монтажного стола: перед ним сотни кадров, и задача — не снять идеальный фильм, а **собрать первый черновой монтаж**, чтобы увидеть, где история теряется.

Идеальный монтаж никогда не начинается с первого дубля. Сначала — rough cut: всё на место, даже если неровно. Потом режиссёр смотрит, что цепляет, а что провисает.

Твой pipeline сегодня — это rough cut. Поймай одно место, которое «провисло» при первом прогоне — именно оно станет точкой роста во второй версии.

</Phase>

<Phase type="concept">

## Полный pipeline: URL → Insights

Это рабочий промпт который можно использовать сразу:

```
Ты — аналитический агент. Твоя задача — извлечь максимальную пользу из контента.

URL для анализа: [вставь ссылку]

Задачи:
1. Прочитай страницу полностью
2. Выдели 5–7 применимых идей (конкретных, не общих фраз)
3. Определи 5–7 тем для дальнейшего изучения
4. Найди 2–3 лучших цитаты или факта

Формат вывода:
## Ключевые идеи
[список с пояснениями]

## Темы для изучения
[список с коротким контекстом]

## Запомнить
[цитаты или факты]

Сохрани результат в файл ideas.md
```

## Расширенный pipeline: несколько источников

Можно анализировать сразу несколько URL параллельно:

```
Проанализируй три статьи по теме [тема]:
1. [URL 1]
2. [URL 2]
3. [URL 3]

Для каждой: 3 идеи + 1 лучшая цитата.
Потом: найди общее между ними и противоречия.
Сохрани в research-[тема].md
```

</Phase>

<Phase type="practice">

Запусти свой pipeline прямо сейчас:

1. Возьми URL из реальной задачи (статья, документация, пост)
2. Используй промпт из Концепции выше
3. Получи результат
4. Оцени: что понравилось? что можно улучшить?

Сохрани результат в `my-experiments/u3-pipeline-result.md`

Сохрани промпт в `my-experiments/u3-pipeline-prompt.md` — это твой первый переиспользуемый pipeline.
