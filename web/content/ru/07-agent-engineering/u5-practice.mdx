<Phase type="reflection">

Хороший спец агента отвечает на эти вопросы **до** написания кода:

| Вопрос | Что в ответе |
|---|---|
| **Trigger** | Что запускает агента? (cron / webhook / событие / ручной запуск) |
| **Inputs** | Что приходит? (формат, объём, источник) |
| **Steps** | Из каких узлов состоит pipeline? |
| **AI vs Code** | Какие узлы LLM, какие — код (см. u2)? |
| **Outputs** | Куда пишется результат? |
| **Failure modes** | Что может сломаться, что делать? |
| **Observability** | Где смотреть что произошло? |
| **Cost ceiling** | Сколько максимум денег в день? |
| **Human-in-loop** | Где нужна твоя проверка? |

Если хоть один вопрос без ответа — рано писать код.

</Phase>

<Phase type="concept">

**Шаблон спецификации агента**

```markdown
# Agent: [имя]

## Цель
[Одно предложение: что делает агент, какую боль решает]

## Trigger
- Тип: cron / webhook / event / manual
- Параметры: [например, каждые 5 минут / при метке Gmail / при пуше в репо]

## Pipeline
1. [Тип узла] — [что делает] — [in → out]
2. [Тип узла] — [что делает] — [in → out]
3. ...

## AI узлы
Список узлов где используется LLM:
- Узел X: модель [provider/model], промпт в `prompts/x.md`
- ...

## Tools / APIs
- Gmail API (scope: ...)
- Notion API (workspace: ...)
- ...

## Storage
- Куда пишет результат
- Идемпотентность: как избегаем повторов

## Observability
- Trace: Langfuse / OTel / stdout логи
- Alerting: куда уходит ошибка (Telegram / email / Slack)

## Failure modes
| Сценарий | Поведение |
|---|---|
| LLM 500 | Retry 3× с exp backoff, потом fallback модель |
| Bad input | Лог + Telegram alert + skip |
| Storage full | Hard fail + alert |

## Cost
- LLM calls в день: ~N
- Estimated $: ~$X
- Hard cap: $Y (через провайдер бюджеты)

## Human-in-loop
- Где: [шаг N]
- Что подтверждаешь
- Как: [Telegram сообщение / Notion task / email]
```

**Реальный пример заполненной спеки** — упрощённая версия ENERV ingest:

```markdown
# Agent: ENERV Gmail Ingest

## Цель
Письма с тегом `AI Ingest` автоматически попадают в правильную папку
Obsidian-vault как .md файлы.

## Trigger
- Тип: poll
- Параметры: Gmail API каждую минуту, фильтр `label:AI Ingest`

## Pipeline
1. Gmail Trigger — забрать письма — () → [emails]
2. Normalize Input — стандартизация полей — [email] → [normalized]
3. Gemini Router — классификация — [normalized] → [folder, filename]
4. Parse Response — валидация JSON — [llm_out] → [structured]
5. ACL Transformer — генерация .md — [normalized + structured] → [md]
6. Source Check — дедуп — [md] → [md или skip]
7. Write — запись файла — [md] → /home/node/obsidian/{folder}/{filename}.md
8. Mark Ingested — пометка Gmail — [email_id] → ()
9. Remove Ingest Label — снятие триггера — [email_id] → ()

## AI узлы
- Gemini Router — gemini-flash, промпт `gemini-router.md`
- ACL Transformer — gemini-flash, промпт `acl-transformer.md`

## Tools
- Gmail API (modify scope)
- File system (volume mount /home/node/obsidian)

## Storage
- Obsidian vault на host volume
- Idempotency: labels `AI Ingest` снимается, добавляется `AI-Ingest-Done`

## Observability
- n8n executions log
- Langfuse traces для обоих LLM-узлов

## Failure modes
| Сценарий | Поведение |
|---|---|
| Gemini 500 | n8n retry 3× с exp backoff |
| Битый JSON | Лог + письмо оставляется с `AI Ingest` |
| Disk full | Email alert (Resend) |

## Cost
- ~10 писем в день × 2 LLM-вызова = 20 calls/day
- Gemini Flash: ~$0.001/call → ~$0.02/day
- Hard cap: $5/month через Vertex AI budget

## Human-in-loop
- Письма помеченные `AI-Ingest-Review` — раз в неделю просматриваю папку
  Atlas/Notes/Sources вручную, чтобы поймать misclassification
```

Это **на бумаге**. Кода ещё нет. Но любой человек — включая меня самого через полгода — может это прочитать и понять что делает агент.

</Phase>

<Phase type="practice">

Заполни шаблон спеки выше для **твоей** задачи из юнита 1.

Ограничения:
- Не пиши «как будто я уже всё знаю». Если не знаешь — напиши `[?]` и оставь вопрос.
- Один файл `my-experiments/my-agent-spec.md`
- Минимум 1 страница, максимум 3
- Покажи кому-то (партнёру, другу, ChatGPT) и спроси: «понятно что делает агент?»

**Sanity check перед концом юнита:**

- [ ] Trigger четкий
- [ ] Pipeline 3+ шагов
- [ ] Хотя бы 1 LLM-узел и 1 не-LLM-узел
- [ ] Описан хотя бы 1 failure mode
- [ ] Понятно сколько ~стоит в день
- [ ] Указано где смотреть трейсы

Если все 6 галочек — у тебя есть спецификация по которой можно начинать строить.
