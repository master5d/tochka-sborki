<Phase type="activation">

Вспомни последнюю задачу, где ты сел писать промпт — и через 10 минут понял, что **один промпт это не решит**.

Может быть:
- Нужно скачать данные из 5 источников
- Прогнать их через анализ
- Сохранить в нескольких местах
- Запустить по расписанию
- Уведомить тебя если что-то пошло не так

Это не «плохой промпт». Это **другая задача класса**.

Запиши:
- Какая задача у тебя сейчас «висит», потому что одного промпта мало?
- Сколько шагов в ней?
- Сколько разных систем участвует?

</Phase>

<Phase type="reflection">

Карпатый и комьюнити в 2026-м провели чёткую границу:

**Vibe coding** = опускает порог входа до нуля. Любой нонкодер за вечер собирает MVP через ChatGPT. Это про **прототип**.

**Agent engineering** = поднимает потолок профессии. Дисциплина координирования множества мощных, но непредсказуемых агентов так, чтобы это работало надёжно. Это про **продакшн**.

Цитата которую стоит выписать:

> *Можно отдать на аутсорс мышление, но нельзя — понимание.*

Vibe coder пишет один промпт и смотрит что вышло.
Agent engineer **проектирует систему**, в которой 5 промптов работают по очереди, проверяют друг друга, восстанавливаются после ошибок, и пишут трейсы — чтобы он мог разобраться **что именно пошло не так** через две недели.

Подумай:
- Что в твоей задаче — vibe? (быстрый прототип, одна попытка)
- Что — agent engineering? (надёжность, повтор, восстановление)

</Phase>

<Phase type="concept">

**Признаки того, что ты упёрся в потолок vibe coding:**

| Симптом | Что это значит |
|---|---|
| Промпт работает 7 раз из 10 | Нужна верификация и retry |
| Нужны данные из 3+ источников | Нужен pipeline, не промпт |
| Задача повторяется каждый день/час | Нужен trigger + scheduling |
| Ошибка стоит денег/доверия | Нужна observability и human-in-loop |
| Хочешь дать запустить другому | Нужна спецификация и UI |

Если хоть один пункт — добро пожаловать в **агентский инжиниринг**.

**Три эры программирования** (Karpathy):

```
Software 1.0  → люди пишут код
Software 2.0  → люди пишут данные, ML модели «обучаются»
Software 3.0  → люди пишут промпты, нейросети «исполняют»
```

Vibe coder живёт в 3.0 и пользуется одним нейрон-«работником».
Agent engineer живёт в 3.0 и **оркестрирует команду** нейрон-работников. Плюс инструменты (MCP), плюс память, плюс recovery.

</Phase>

<Phase type="practice">

Возьми задачу из активационной фазы. Ответь письменно:

1. **Сколько шагов** в задаче (включая загрузку данных, обработку, сохранение, уведомление)
2. **Сколько раз в неделю** ты бы её повторил, если бы она работала автоматически
3. **Сколько часов** в неделю сейчас уходит на её ручное выполнение
4. **Цена ошибки** — что плохого случится если шаг 3 сорвётся

Если ответы: ≥3 шагов, ≥3 раза в неделю, ≥1 час, ошибка ощутима — этот модуль ровно про твою задачу.

Сохрани в `my-experiments/agent-task-scope.md`.

</Phase>
