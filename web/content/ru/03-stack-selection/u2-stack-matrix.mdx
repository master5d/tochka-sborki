## Trade-offs честно

Никакой стек не «лучше» — у каждого свои стороны:

| Аспект | Claude Code | Sovereign | Cloud-OSS | Behind-GFW |
|---|---|---|---|---|
| **Качество out-of-box** | 9/10 | 7/10 | 7/10 | 6/10 |
| **Настройка** | 5 мин | пол-дня | 30 мин | 1 час |
| **Цена/мес** | $20+ | $0 + железо | $0–5 | $0 |
| **Privacy** | низкая | максимальная | средняя | средняя |
| **Скорость** | средняя | высокая (local) | переменная | очень высокая (Cerebras) |
| **Vendor lock-in** | Anthropic | нет | OpenRouter/провайдер | Cerebras/Google |
| **Доступ в Китае** | требует VPN | работает | переменно | работает |

## Decision tree (если всё ещё в сомнениях)

```
Ты в Китае без VPN?
├── ДА → Behind-GFW
└── НЕТ
    │
    Бюджет $20+/мес и не критична приватность?
    ├── ДА → Claude Code
    └── НЕТ
        │
        Есть GPU или Mac M2+?
        ├── ДА + важна автономность → Sovereign
        └── НЕТ → Cloud-OSS
```

<Phase type="reflection">

Заметь: **«гибридные»** варианты тоже валидны. Многие SOVERN-практики используют:
- Claude Code как **primary** для тонких задач
- Local llama-server как **floor** когда нет интернета
- Cerebras как **fast-path** для бойлерплейта (бесплатно, 1000+ TPS)

LiteLLM gateway делает это бесшовным — для агента это «один OpenAI-compatible endpoint», а под капотом — fallback массив.

Это и есть **agent-agnostic мышление**: ты управляешь стеком, а не привязываешься к одной тулзе.

</Phase>
