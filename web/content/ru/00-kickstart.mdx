## 🧩 Что такое vibe coding?

Простыми словами: **ты говоришь — AI пишет код**.

### Старая разработка (Software 1.0 / 2.0)

Чтобы сделать приложение, нужна была команда:

| Роль | Что делает |
| --- | --- |
| **Product Manager** | Решает, что делать и зачем |
| **UI/UX Designer** | Рисует дизайн экранов |
| **Frontend Developer** | Делает интерфейс (кнопки, экраны) |
| **Backend Developer** | Пишет логику на сервере |
| **QA Engineer** | Тестирует и ловит баги |
| **DevOps** | Выкладывает приложение в интернет |

Учиться на это — **3–5 лет**.

### Vibe coding (Software 3.0)

**Ты — PM + QA + DevOps. AI — всё остальное.**

```
Ты:   решаешь ЧТО нужно  →  тестируешь результат  →  запускаешь в мир
AI:   проектирует  →  пишет код  →  чинит баги  →  подключает базы
```

Учиться — **от пары дней до пары недель**. Остальное приходит с практикой.

## 🗺️ Карта AI-инструментов

Рынок делится на 4 квадранта. Полезно знать, где что находится:

| Квадрант | Что делает | Примеры |
| --- | --- | --- |
| 🟦 **Zero-code builders** | Промпт → готовое приложение за минуты | Lovable, Bolt, Replit, Bubble |
| 🟪 **AI dev platforms** *(здесь Claude Code)* | Глубокая работа с кодом, файлами, проектом | **Claude Code**, Codex |
| 🟨 **Design-to-code** | Дизайн → код | Figma Make, Stitch, Framer |
| 🟩 **AI coding assistants** | Помощники внутри IDE | Cursor, Windsurf |

**Claude Code** — самый мощный тир. Ты получаешь не просто чат, а агента, который:
- Видит твою файловую систему
- Запускает команды в терминале
- Помнит контекст между сессиями (через `CLAUDE.md`)
- Расширяется MCP-серверами, Skills, Hooks

> 💡 Для простой лендинг-страницы хватит Lovable. Для серьёзной работы с кодом, автоматизаций и собственных агентов — Claude Code. **Этот курс — про второе.**

## 📚 Минимальный словарь

Не обязательно знать наизусть — достаточно узнавать, когда встретишь:

### Про AI

| Термин | Суть |
| --- | --- |
| **LLM** | Large Language Model — "мозг" AI (Claude, GPT и т.д.) |
| **Токен** | Кусочек слова; LLM считает стоимость в токенах |
| **Промпт** | Текст, который ты даёшь LLM |
| **Контекст** | Всё, что LLM "помнит" в рамках сессии |
| **Агент** | LLM + инструменты + память + цикл действий |
| **Галлюцинация** | Когда LLM уверенно врёт (выдумывает факты) |

### Про разработку

| Термин | Суть |
| --- | --- |
| **Frontend** | То, что видит пользователь (кнопки, экраны, анимации) |
| **Backend** | То, что крутится на сервере (логика, авторизация, вычисления) |
| **Database** | Где хранятся данные (Supabase, Postgres, SQLite) |
| **API** | Способ одним программам говорить с другими |
| **Deployment** | Выкладка приложения в интернет, чтобы им могли пользоваться |
| **Git** | История изменений кода; возможность откатиться |
| **CLI** | Command Line Interface — работа через терминал (чёрное окно) |

### Про Claude Code (забежим вперёд)

| Термин | Суть |
| --- | --- |
| **CLAUDE.md** | Файл с контекстом проекта, который Claude читает в начале |
| **MCP** | Model Context Protocol — способ добавить Claude новые инструменты |
| **Skill** | Модульная "суперспособность" для Claude (напр. `create-pdf`) |
| **Hook** | Автоматическое действие на событие (напр. "перед каждым коммитом") |
| **Subagent** | Отдельный агент для конкретной задачи (напр. ревью кода) |

## 🧠 Твоя новая роль

В vibe coding ты играешь три партии одновременно:

```
1. 🎯  Project Manager  →  формулируешь, ЧТО нужно получить
2. 🧪  QA Engineer      →  проверяешь, что AI сделал правильно
3. 🚀  DevOps           →  запускаешь результат в реальный мир
```

**AI** закрывает остальные роли: дизайн, frontend, backend, отладка. Но без тебя AI не знает, куда бежать — у него нет твоих целей, вкуса, понимания аудитории.

> 💡 **Главный парадокс vibe coding:** чем лучше ты умеешь *описывать*, тем меньше нужно уметь *кодить*.

## ⚡ Частые страхи и ответы

**«Я сломаю что-нибудь важное»**
→ Claude Code спрашивает разрешение перед опасными действиями. Плюс Git всегда позволяет откатиться.

**«Я не пойму технических терминов»**
→ Claude сам объясняет. Спрашивай «объясни простым языком» — это рабочая команда.

**«Мне нужно сначала выучить Python/JavaScript»**
→ Нет. Достаточно понимать, *что* ты строишь. Язык AI выберет сам.

**«А если AI ошибётся?»**
→ Он ошибётся. Meeting 3 учит замечать и исправлять. Ошибки — часть процесса, не провал.

**«С чего мне вообще начать?»**
→ Meeting 1 и Meeting 2. Там пошаговая установка и первый рабочий проект.
