<Phase type="concept">

## Любая автоматизация = Trigger → Action

Это фундаментальный mental model:

| Trigger | Action |
|---------|--------|
| Будильник 6:00 | Телефон звонит |
| Оплата прошла | Email + SMS клиенту |
| Ты даёшь URL агенту | Агент скрапит → анализирует → сохраняет результат |

### Три вопроса перед построением pipeline

1. **Trigger:** что запускает процесс? (ты вручную, schedule, новый файл, webhook)
2. **Action:** что должно произойти? (scrape → analyze → save → notify)
3. **Condition:** когда НЕ запускать? (дубли, rate limits, ошибки)

### Наш pipeline сегодня

```
Trigger:  Ты даёшь URL статьи
   ↓
Action 1: Firecrawl скрапит страницу → получает чистый текст
   ↓
Action 2: Claude анализирует текст
   ↓
Action 3: Результат сохраняется в Markdown-файл
```

Это ручной trigger. Но принцип тот же — что для Zapier, что для Claude Code Hooks.

</Phase>
