<Phase type="activation">

Какие повторяющиеся информационные потоки у тебя есть?

Например:
- Каждое утро читаешь определённые источники
- После встреч пишешь summary
- Раз в неделю собираешь отчёт из разных источников

Выбери один такой поток.

</Phase>

<Phase type="reflection">

Если ты делаешь это каждую неделю — ты уже знаешь алгоритм.

Алгоритм можно описать как: «Когда [событие] → делаю [действие] → получаю [результат]».

Это именно та структура, которую понимает автоматизация.

</Phase>

<Phase type="concept">

## Три типа pipeline

### 1. Ручной pipeline (сегодня)
Ты запускаешь вручную когда нужно. Идеально для нерегулярных задач.

```
Ты → [URL/файл] → Claude → результат
```

### 2. Schedule pipeline
Запускается по расписанию. Подходит для регулярных задач.

```
Каждое утро в 8:00 → собрать RSS → анализировать → отчёт в Slack
```

Инструменты: n8n, Make.com, GitHub Actions, Claude Code Hooks.

### 3. Event-driven pipeline
Запускается когда что-то происходит.

```
Новый файл в папке → автоматически транскрибировать → суммировать
Новый email → классифицировать → черновик ответа → на согласование
```

## Инструменты для pipeline

| Инструмент | Уровень | Для чего |
|------------|---------|---------|
| Claude (chat) | Начало | Разовые задачи, тестирование |
| Claude Code | Средний | Работа с файлами, проектами |
| Make.com / n8n | Продвинутый | Автоматизация без кода |
| Claude Code Hooks | Экспертный | Событийная автоматизация |

> 💡 Начни с ручного pipeline. Как только убедишься что алгоритм работает — автоматизируй trigger.

</Phase>
