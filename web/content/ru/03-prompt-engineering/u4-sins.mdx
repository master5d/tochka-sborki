## Пять грехов промптинга

### 1. Context Rot — «протухание контекста»

Чем длиннее разговор, тем хуже модель работает с ранней информацией.

**Решение:** структурируй информацию по приоритету, клади важное в начало и повторяй в конце.

### 2. Lost in the Middle — середина теряется

Модель переоценивает начало и конец контекста. Критичное в середине игнорируется.

```
❌ Плохо: [Введение] → [КРИТИЧНО — в середине] → [Заключение]
✅ Хорошо: [КРИТИЧНО — в начале] → [Детали] → [Повтор КРИТИЧНОГО]
```

### 3. Дистракторы — лишнее отвлекает

Похожие фрагменты в контексте сбивают модель с правильного ответа.

```
❌ Плохо: Даёшь 5 файлов, 2 из которых — старые версии
✅ Хорошо: Только релевантные файлы + «Игнорируй старые версии»
```

### 4. Смешение контекста и промпта

Контекст (кто ты, что происходит) и инструкция (что конкретно сделать) — разные вещи.

```
Контекст = фоновая информация (стабильна)
Промпт   = конкретная задача (для этой сессии)
```

Отделяй их явно — через заголовки Markdown или XML-теги.

### 5. Брань снижает качество

Это реальный эффект: агрессивный тон в промпте ухудшает результаты.

```
❌ «Это полностью неправильно! Переделай срочно!»
✅ «Результат не подошёл. Попробуй без рекурсии.»
```

</Phase>
