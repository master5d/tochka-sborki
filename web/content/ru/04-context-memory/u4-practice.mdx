<Phase type="activation">

У тебя теперь есть CLAUDE.md, TODO.md, AGENTS.md.

Но чего в них не хватает? Что ты ещё не написал, хотя хотел бы, чтобы Claude это знал?

</Phase>

<Phase type="reflection">

Хорошая система памяти — это не разовая настройка. Это живая система.

Правило: если тебе приходится объяснять одно и то же дважды — это сигнал добавить в CLAUDE.md.

Правило: если Claude сделал что-то не так и ты его исправил — это сигнал добавить в AGENTS.md.

</Phase>

<Phase type="concept">

## Когда обновлять что

| Событие | Где обновить |
|---------|-------------|
| Изменилась архитектура | CLAUDE.md |
| Добавили новый стек | CLAUDE.md |
| Завершена задача | TODO.md |
| Агент сделал ошибку, которую ты исправил | AGENTS.md |
| Небольшое уточнение в середине сессии | `/btw Новая деталь...` |

## Техники для длинных сессий

### Сжатие контекста

В начале новой сессии или когда контекст заполнился:

```
Кратко резюмируй что было сделано и в каком состоянии проект.
Используй формат: ✅ Сделано / ❌ Осталось / ⚠️ Проблемы
```

### Приоритизация информации

```markdown
## ⚠️ КРИТИЧНО (всегда держи в фокусе)
[Самое важное для текущей задачи]

## 📋 КОНТЕКСТ (полезно знать)
[Дополнительные детали]
```

### Параллельные агенты для больших задач

Для независимых подзадач можно запустить несколько агентов:

```
Агент 1: Найди все баги в модуле Auth
Агент 2: Напиши тесты для модуля Payment
Агент 3: Обнови документацию API

Потом объедини результаты.
```

</Phase>
