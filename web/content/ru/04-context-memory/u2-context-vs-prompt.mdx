<Phase type="activation">

Представь, что ты нанимаешь нового сотрудника в команду.

Что ты ему объясняешь в первый день? (контекст о компании, роль, цели, правила)

А что ты пишешь ему в конкретном задании? (что сделать прямо сейчас)

Это разные вещи. Почему?

</Phase>

<Phase type="reflection">

Контекст — стабильный фон. Задание — конкретная инструкция на этот момент.

Проблема в том, что большинство людей смешивают их в одном промпте — каждый раз заново объясняют всё. Это расходует контекст, замедляет работу и создаёт путаницу.

</Phase>

<Phase type="concept">

## Разделение контекста и промпта

```
Контекст  = информация о мире (цели, роли, правила, данные, история)
Промпт    = конкретная инструкция на этот запрос

«Напиши письмо» = ПРОМПТ (задание)

«Я основатель стартапа, пишу инвестору, стиль лаконичный и уверенный,
цель: договориться о встрече в среду вечером» = КОНТЕКСТ
```

### Как разделять явно

```markdown
## КОНТЕКСТ
[Роль, цель, ограничения, данные — то что стабильно]

## ЗАДАЧА
[Конкретная инструкция для этого запроса]
```

Или через XML-теги:

```xml
<context>
Ты — специалист по code review. Проект использует Python 3.11.
</context>

<task>
Проверь этот код на ошибки безопасности.
</task>

<code>
[Код здесь]
</code>
```

### Где хранить стабильный контекст

| Что | Где хранить |
|-----|-------------|
| Правила проекта, стек | CLAUDE.md в папке проекта |
| Личные предпочтения | ~/.claude/CLAUDE.md |
| Текущие задачи | TODO.md |
| Инструкции для агента | AGENTS.md |

</Phase>

<Phase type="practice">

Возьми любую задачу из работы. Разбей её на два блока:

```markdown
## КОНТЕКСТ
[Кто ты, что за проект, стек, ограничения]

## ЗАДАЧА
[Только конкретная инструкция]
```

Сравни результат с тем, как ты обычно формулируешь промпт.

Сохрани в `my-experiments/u2-context-vs-prompt.md`

</Phase>
