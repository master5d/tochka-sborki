## Five sins of prompting

### 1. Context Rot

The longer the conversation, the worse the model handles early information.

**Fix:** structure information by priority, put the important stuff at the start and repeat it at the end.

### 2. Lost in the Middle

The model over-weights the beginning and end of the context. Critical stuff in the middle gets ignored.

```
❌ Bad:  [Intro] → [CRITICAL — in the middle] → [Conclusion]
✅ Good: [CRITICAL — up front] → [Details] → [Repeat CRITICAL]
```

### 3. Distractors — extra stuff pulls attention

Similar fragments in the context throw the model off the correct answer.

```
❌ Bad:  Drop 5 files, 2 of which are old versions
✅ Good: Only relevant files + "Ignore old versions"
```

### 4. Mixing context and prompt

Context (who you are, what's going on) and instruction (what to do specifically) are different things.

```
Context = background info (stable)
Prompt  = the specific task (for this session)
```

Separate them explicitly — with Markdown headings or XML tags.

### 5. Cursing kills quality

This is a real effect: aggressive tone in the prompt makes results worse.

```
❌ "This is completely wrong! Redo it now!"
✅ "That didn't work. Try without recursion."
```
