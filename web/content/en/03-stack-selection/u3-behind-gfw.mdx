<Phase type="activation">

Picture a situation: between you and the tool you need — a closed border.

Don't picture it as a "list of blocked domains." Picture it as **a smuggler's route**: an experienced courier doesn't weep at locked gates — they know which paths stay open, where the cache is hidden, how to move cargo legally through the neighbouring crossing.

Run it in your head: what's your "cargo" (the AI tool you need), where's the "border" (GFW, corporate firewall, no card access), and — **which routes are actually open**?

Here's the map of both sides:

**Doesn't work in China without VPN:**
- `claude.ai`, `console.anthropic.com` — Anthropic
- `chatgpt.com`, `platform.openai.com` — OpenAI
- `cursor.com`, most AI-IDEs
- `huggingface.co` — models and datasets (blocked since 2023)
- GitHub partially, npm registry (via mirrors)

**Works out of the box:**
- `cerebras.ai` — Cerebras Cloud Inference (US-based, not in China blocklist)
- `aistudio.google.com` — Google Gemini Flash free tier
- `deepseek.com`, `qianwen.aliyun.com` — Chinese providers
- `openrouter.ai` — partially, depends on model
- Local models (after download) — forever

</Phase>

<Phase type="reflection">

Picture a village herbalist 🌿, miles from the nearest pharmacy. No QR-coded pills — but they know which roots in this forest break a fever, where the right bark grows, how to combine what's local into something a drugstore never imagined.

Notice the same paradox: **the strongest open models in 2026 are Chinese**. Qwen (Alibaba), DeepSeek-Coder, Yi (01.AI), GLM (Tsinghua) — all top-tier on benchmarks.

If you're in China — you actually have **extended** access to OSS models via local providers (Qianwen, Volcengine, DeepSeek API). Card barriers are usually lower — `WeChat Pay` / `Alipay` integrated.

This unit isn't about bypassing — it's about **building a full agentic workflow from local herbs**. No pharmacy required.

## "Behind-GFW" architecture

```
┌─────────────────────────────────────────────────────────┐
│  Frontend: Hermes / Aider / Cline / Continue.dev        │
├─────────────────────────────────────────────────────────┤
│  Router: LiteLLM (localhost:4000)                       │
│  ┌────────────────────────────────────────────────────┐ │
│  │ fast-path:  Cerebras Qwen-3-235B (1000 TPS)        │ │
│  │ balanced:   Gemini 2.5 Flash (via AI Studio)       │ │
│  │ chinese:    DeepSeek-Coder API / Qwen API          │ │
│  │ floor:      local llama-server (Qwen3-Coder-30B)   │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

## Step 1: Cerebras (main free tier)

1. Go to [cerebras.ai/cloud-inference](https://cerebras.ai/cloud-inference) — works without VPN
2. Sign up with email (no card needed)
3. Free tier: ~1M tokens/day, 1000+ TPS on Qwen-3-235B
4. Grab the API key → save in `~/.config/sovern/.env`:

```bash
# ~/.config/sovern/.env
CEREBRAS_API_KEY=csk-xxxxx
```

## Step 2: Google Gemini Flash (backup for longer tasks)

1. `aistudio.google.com` — works from mainland via mainland Google domain
2. Free tier: 15 RPM / 1500 RPD on Gemini 2.5 Flash
3. If AI Studio is blocked — there are proxy-equivalents through Volcengine/Alibaba

## Step 3: LiteLLM gateway

"One endpoint for all providers". Installed locally:

<OsToggle label="Commands for:" />

<OsBlock os="mac">

```bash
pip install 'litellm[proxy]'
mkdir -p ~/.config/litellm
```

</OsBlock>

<OsBlock os="windows">

```powershell
pip install 'litellm[proxy]'
New-Item -ItemType Directory $HOME\.config\litellm -Force
```

</OsBlock>

Minimal config in `~/.config/litellm/config.yaml`:

```yaml
model_list:
  - model_name: fast-pool
    litellm_params:
      model: cerebras/qwen-3-235b-a22b
      api_key: os.environ/CEREBRAS_API_KEY

  - model_name: balanced
    litellm_params:
      model: gemini/gemini-2.5-flash
      api_key: os.environ/GOOGLE_API_KEY

  - model_name: local-floor
    litellm_params:
      model: openai/qwen-3-coder
      api_base: http://localhost:8080/v1
      api_key: not-needed

router_settings:
  routing_strategy: simple-shuffle
  fallbacks:
    - fast-pool: ["balanced", "local-floor"]
    - balanced: ["local-floor"]
```

Run it:

```bash
litellm --config ~/.config/litellm/config.yaml --port 4000
```

Now every agent can point at `http://localhost:4000` as an "OpenAI-compatible API".

## Step 4: Local fallback (llama-server)

For when every cloud goes dark at the same time (it happens):

```bash
# Download model (via HuggingFace mirror)
# in China: https://hf-mirror.com replaces huggingface.co
wget https://hf-mirror.com/Qwen/Qwen3-Coder-30B-A3B-Instruct-GGUF/resolve/main/Qwen3-Coder-30B-A3B-Instruct-Q4_K_M.gguf

# Run llama-server (needs llama.cpp)
llama-server -m Qwen3-Coder-30B-A3B-Instruct-Q4_K_M.gguf --port 8080
```

> 💡 On Mac M-series with 16GB RAM, Q4_K_M runs at ~30 t/s — tolerable. On 32GB+ — comfortable.

## Step 5: Wire up an agent

Any OpenAI-compatible agent (Aider, Cline, Continue.dev, Hermes) plugs in the same way:

```bash
# Aider
aider --model openai/fast-pool --openai-api-base http://localhost:4000

# Cline (VS Code) — in settings:
# OpenAI Compatible: http://localhost:4000
# Model: fast-pool
```

Done. Full agentic workflow without Anthropic, without OpenAI, without VPN.
