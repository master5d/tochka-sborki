## Honest trade-offs

No stack is "best" — each has its angle:

| Aspect | Claude Code | Sovereign | Cloud-OSS | Behind-GFW |
|---|---|---|---|---|
| **Out-of-box quality** | 9/10 | 7/10 | 7/10 | 6/10 |
| **Setup time** | 5 min | half a day | 30 min | 1 hour |
| **Price/month** | $20+ | $0 + hw | $0–5 | $0 |
| **Privacy** | low | maximum | medium | medium |
| **Speed** | medium | high (local) | variable | very high (Cerebras) |
| **Vendor lock-in** | Anthropic | none | OpenRouter/provider | Cerebras/Google |
| **China access** | needs VPN | works | variable | works |

## Decision tree (if still unsure)

```
Are you in China without VPN?
├── YES → Behind-GFW
└── NO
    │
    Budget $20+/mo and privacy not critical?
    ├── YES → Claude Code
    └── NO
        │
        Have a GPU or Mac M2+?
        ├── YES + autonomy matters → Sovereign
        └── NO → Cloud-OSS
```

<Phase type="reflection">

Picture an orchestra with no conductor 🎻. Each musician is brilliant — but without coordination, you get noise, not a symphony.

Notice: that's exactly what happens when stacks aren't coordinated. Many SOVERN practitioners blend several instruments into one sound:
- Claude Code as **primary** for nuanced tasks
- Local llama-server as **floor** when offline
- Cerebras as **fast-path** for boilerplate (free, 1000+ TPS)

LiteLLM gateway is the **conductor** — seamless to the agent, a full fallback score underneath.

Run it in your head: in this orchestra, is your stack the soloist or the section?
