<Phase type="activation">

Before picking a tool — look at your **real constraints**. Answer 5 questions (in your head or in `my-experiments/u1-constraints.md`):

1. **Budget:** how much are you actually willing to pay for AI tools per month?
   - $0 / under $5 / under $20 / under $50 / no limit

2. **Access:** where are you?
   - Mainland China without VPN / GDPR country / US / other

3. **Hardware:** what do you have?
   - Laptop without GPU / Mac M-series / PC with discrete NVIDIA / remote server

4. **Privacy:** how important is it that code never leaves your machine?
   - Doesn't matter / preferred / critical (NDA, healthcare, infra)

5. **Setup tolerance:** how many hours are you willing to spend on initial setup?
   - 15 min / 1 hour / a whole evening / several days

</Phase>

<Phase type="reflection">

Most AI courses start with a tool — "install Claude Code", "open ChatGPT". Then they wonder why half the students drop off at lesson two: one has no card for API, the second has `anthropic.com` blocked, the third's corporate VPN refuses Cursor.

**Context first, then tool.** This rule applies to any agentic stack.

Your set of answers already nearly dictates the choice:

| If you have… | …you probably need |
|---|---|
| $20+ budget, no constraints, minimal setup | **Claude Code** — paid but convenient |
| $0 budget, hardware on hand, autonomy matters | **Sovereign** (per SOVERN) — Hermes + local models |
| $0–5 budget, no hardware, access exists | **Cloud-OSS** — Cline + free-tier clouds |
| China, no VPN | **Behind-GFW** — Cerebras + Gemini Flash + local |

No "right" answer — just the stack that fits your situation.

</Phase>

<Phase type="concept">

## Why not "just Claude Code"

This course was originally built around Claude Code — a strong, convenient tool. But it:

- **Costs money** — minimum ~$20/mo on Sonnet
- **Requires access** to `claude.ai` and the Anthropic API. In China without VPN — unavailable
- **Vendor lock-in** — your prompts, MCP configs, hooks are tied to the Anthropic ecosystem
- **Telemetry** — every request goes to Anthropic. Not all corp / gov environments allow that

For most students that's fine. But if even one of the four bullets above applies to you, Claude Code is not your fit. And you're not "worse" — **agentic AI is no longer a single-tool field**, there are open alternatives.

## What "stack" means here

In this course, a **stack** = combination of three layers:

```
┌─────────────────────────────────────────────┐
│  Frontend / Agent UI                        │  ← Claude Code, Cline, Aider, Hermes
├─────────────────────────────────────────────┤
│  Routing / Gateway                          │  ← LiteLLM, OpenRouter, direct API
├─────────────────────────────────────────────┤
│  Model provider (where intelligence lives)  │  ← Anthropic, Cerebras, Groq, local GGUF
└─────────────────────────────────────────────┘
```

Every layer is swappable. For example:
- **Claude Code → LiteLLM → Cerebras Qwen-3-235B** — Claude Code UI with an open model
- **Aider → direct OpenRouter → DeepSeek-Coder** — classic OSS workflow
- **Hermes → LiteLLM with fallback array** — SOVERN pattern

This module teaches you to **think about the whole stack** rather than "knowing one tool".
