<Phase type="reflection">

Automation isn't "set it and forget it." It's an iterative process.

The first version shows you where the bottleneck is. The second version removes it. The third becomes reliable.

The question isn't "does it work perfectly?" — it's "is it useful enough to keep going?"

</Phase>

<Phase type="concept">

## Automation map: what to delegate next

| Task type | Automatable | Pipeline example |
|------------|-----------------|----------------|
| Information gathering | Easy | URL → summary → file |
| Content analysis | Easy | Text → insights → report |
| Draft emails | Medium | Email → context → draft |
| Data processing | Medium | Table → analysis → viz |
| Meetings | Medium | Audio → transcript → tasks |
| Decision-making | Hard | Needs a human |

## Next steps: from manual to automatic

Once the manual pipeline works:

1. **Add conditions:** "if the article is under 500 words — skip"
2. **Add a schedule:** run every morning at 8:00
3. **Add notifications:** send results to Telegram or email
4. **Connect to other tools:** Google Docs, Notion, Slack

Tools for the next level: Make.com, n8n, Claude Code Hooks (Meeting 6).

</Phase>
