<Phase type="reflection">

If you do this every week — you already know the algorithm.

The algorithm can be described as: "When [event] → I do [action] → I get [result]".

That's exactly the structure automation understands.

</Phase>

<Phase type="concept">

## Three pipeline types

### 1. Manual pipeline (today)
You run it manually when needed. Ideal for irregular tasks.

```
You → [URL/file] → Claude → result
```

### 2. Scheduled pipeline
Runs on a schedule. Good for regular tasks.

```
Every morning at 8:00 → collect RSS → analyze → report to Slack
```

Tools: n8n, Make.com, GitHub Actions, Claude Code Hooks.

### 3. Event-driven pipeline
Runs when something happens.

```
New file in a folder → auto-transcribe → summarize
New email → classify → draft reply → for approval
```

## Tools for pipelines

| Tool | Level | For what |
|------------|---------|---------|
| Claude (chat) | Beginner | One-off tasks, testing |
| Claude Code | Intermediate | Working with files, projects |
| Make.com / n8n | Advanced | No-code automation |
| Claude Code Hooks | Expert | Event-driven automation |

> 💡 Start with a manual pipeline. Once you've confirmed the algorithm works — automate the trigger.

</Phase>
