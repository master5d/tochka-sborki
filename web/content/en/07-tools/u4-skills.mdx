<Phase type="reflection">

A good reusable prompt is already almost a Skill.

Don't think of this as "saved text." Picture a locksmith who cut a key to fit one specific lock: it slides in without fumbling, on the first move — no trial and error, no friction. A Skill is that key for your task: `/skill-name`, and the agent opens exactly what's needed.

Notice in your head: which lock in your work is still picked by trial and error every time?

</Phase>

<Phase type="concept">

## What is a Skill

A Skill is a Markdown file with instructions that Claude Code runs when you call `/name`.

### Skill file structure

```markdown
---
name: meeting-summary
description: Summarizes a meeting from a transcript or notes
---

# Meeting Summary Skill

## Goal
Create a structured meeting recap.

## Instruction
You are a meeting assistant. Based on the provided data:

1. Pull out the main decisions (3–5 bullets)
2. Identify tasks with owners and deadlines
3. Note open questions

## Output format
### Decisions
- [list]

### Tasks
- [ ] [task] — [owner] — [deadline]

### Open questions
- [list]
```

### Where to store Skills

- `~/.claude/skills/` — personal, work across all projects
- `.claude/skills/` — project-scoped, in git

### How to call

```
/meeting-summary [paste transcript or notes]
```

## Plugin vs Skill

| | Plugin | Skill |
|--|--------|-------|
| What | Package with multiple skills | One specific action |
| How to use | Install once | Call via `/name` |
| Example | `firecrawl@local` | `/firecrawl:firecrawl-cli` |
