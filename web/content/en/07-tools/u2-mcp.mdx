Don't think of this as "connecting tools." Picture a telegraph cable between two distant stations: the radio operator on each end knows Morse — it doesn't matter who's on the other side, the signal goes through. One protocol instead of a thousand bilateral treaties.

<Phase type="concept">

## MCP: three primitives

| Primitive | Controlled by | What it is |
|----------|-------------|---------|
| **Tools** | The model | Functions the agent calls: API requests, writing files |
| **Resources** | The application | Data sources (like GET requests): files, DB records |
| **Prompts** | The user | Templates for optimal usage |

## How to connect an MCP server

In `.claude/settings.json` add:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "your-token"
      }
    }
  }
}
```

## Popular MCP servers

| Server | What it gives you |
|--------|---------|
| `@modelcontextprotocol/server-github` | Issues, PRs, commits, repo files |
| `@modelcontextprotocol/server-filesystem` | Extended file system access |
| `@modelcontextprotocol/server-brave-search` | Web search via Brave |
| Notion MCP | Read and write Notion pages |
| Firecrawl | Web page scraping |

## Common errors (~60% of issues)

- `Server transport closed unexpectedly` — syntax error in the JSON config
- Port conflicts — two servers trying to listen on the same port
- Missing env var — token wasn't passed correctly

Fix: check JSON syntax and tokens before digging deeper.
