<Phase type="activation">

Look at how many AI tools get advertised every single day. ChatGPT, Claude, Cursor, Lovable, Bolt...

**Don't you feel overwhelmed?** How do you figure out what you actually need?

</Phase>

<Phase type="reflection">

Most people try everything at once and quit before getting a result.
To avoid drowning, you need to understand the **quadrants** — which problem each tool actually solves.

</Phase>

<Phase type="concept">

## The map: 4 quadrants of AI tools

| Quadrant | What it does | Examples |
|----------|-----------|---------|
| 🟦 **Zero-code builders** | Prompt → ready app in minutes | Lovable, Bolt, Replit |
| 🟪 **AI dev platforms** | Deep work with code and projects | **Claude Code**, Codex |
| 🟨 **Design-to-code** | Design → code | Figma Make, Stitch |
| 🟩 **AI coding assistants** | Helpers inside the IDE | Cursor, Windsurf |

## Where this course sits

**Claude Code** is quadrant 🟪. It's the "heavy artillery". It doesn't just draw buttons — it manages files, the terminal, and the project's memory.

## Your new role: PM + QA + Conductor

In Software 3.0 the roles have shifted:
1. **Product Manager (YOU):** decide WHAT to build.
2. **AI agent:** designs and writes the code.
3. **QA (YOU):** verify it works.
4. **DevOps (YOU + AI):** ship the project to the world.

</Phase>

<Phase type="practice">

Pick one quadrant that seems most useful to you right now (other than Claude Code).

Find one tool from that quadrant (e.g. head to lovable.dev or bolt.new).

Try to create a "Hello World" there in 1 minute.

Note the feeling: what was easy, and where did you feel a lack of control? Save it to `my-experiments/u2-first-tools-test.md`.

</Phase>
