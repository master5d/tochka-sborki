<Phase type="activation">

When you start a new Claude Code session — what do you re-explain every time?

Who you are? What the project is? Which tech? What the rules are?

Write it down. That's what should live in your memory system.

</Phase>

<Phase type="reflection">

Explaining the same thing every time burns time and tokens.

But there's the opposite problem too: a CLAUDE.md with a thousand lines is worse than one with 100 lines. The longer it is, the worse the agent follows instructions.

You need a system: minimal context with maximum influence on behavior.

</Phase>

<Phase type="concept">

## Three memory files

### CLAUDE.md — project context (the main one)

```markdown
# Project Context

## My role
[Profile, expertise]

## Project
[What this is, why it exists]

## Stack
- Node.js 18, TypeScript, React

## Structure
- /src — code
- /tests — tests

## How to run
npm install && npm run dev

## What to delegate
- Writing code, tests, docs

## What I decide myself
- Architecture, final review
```

**Limit: ~200 lines.** More than that and instruction-following degrades.

### TODO.md — current tasks

```markdown
## 🔴 NOW
- [ ] Task 1 (2 h)

## 🟡 NEXT
- [ ] Task 2

## ✅ DONE
- [x] Task 3
```

### AGENTS.md — instructions for the agent

```markdown
## For any task
- Use TypeScript
- Write tests alongside code

## For /explore
- Focus on architecture

## What to avoid
- Don't change architecture without sign-off
```

### MEMORY.md — Claude's auto-memory

Unlike the three files above, Claude **writes MEMORY.md itself**.

What it saves: patterns, fixes, project quirks.
Where it lives: `~/.claude/projects/<hash>/memory/` — local only.
Management: `/memory` — view and edit.

</Phase>

<Phase type="practice">

Create your memory system:

```bash
mkdir -p ~/my-ai-project
cd ~/my-ai-project
touch CLAUDE.md TODO.md AGENTS.md
```

Fill in CLAUDE.md using the template above — use a real project or a task from your work.

Add at least 3 tasks to TODO.md that you actually have right now.

Launch Claude Code and ask: `"What do you know about my project?"`

Save the answer to `my-experiments/u3-memory-test.md`

</Phase>
