<Phase type="concept">

## When to update what

| Event | Where to update |
|---------|-------------|
| Architecture changed | CLAUDE.md |
| New stack added | CLAUDE.md |
| Task completed | TODO.md |
| Agent made a mistake you fixed | AGENTS.md |
| Small mid-session clarification | `/btw New detail...` |

## Techniques for long sessions

### Context compression

At the start of a new session or when context fills up:

```
Briefly summarize what's been done and the state of the project.
Use format: ✅ Done / ❌ Left / ⚠️ Issues
```

### Prioritizing information

```markdown
## ⚠️ CRITICAL (always keep in focus)
[The most important for the current task]

## 📋 CONTEXT (good to know)
[Additional details]
```

### Parallel agents for big tasks

For independent subtasks, run several agents:

```
Agent 1: Find all bugs in the Auth module
Agent 2: Write tests for the Payment module
Agent 3: Update API docs

Then merge the results.
```
